Code - к категории программного кода относят имена переменных, ключевые слова, тексты функции и т.д. Обычно браузеры отображают содержимое тега code как моноширинный текст уменьшенного размера. в отличие от тега pre дополнительные пробелы и переносы текста внутри тега code не учитываются. Поэтому для создания переносов используйте тег br или p. Тег code не считается устаревшим, однако лучших результатов можно достичь при помощи стилей CSS.

address - определяет информацию об авторе документа или статьи. Если тег address расположен внутри тега body, то он представляет информацию об авторе документа. Если тег address расположен внутри тега article, то он представляет информацию об авторе статьи. По умолчанию текст внутри элемента address отображается курсивом. Кроме этого, большинство браузеров добавляют перед элементом и после него разрыв строки. Тег address не должен использоваться для предоставления почтового адреса, если только он не является частью информации об авторе. Как правило, элемент address включается наравне с другой информацией в элемент footer.

Nobr - элемент предотвращает автоматическое перенос текста на несколько строк, что потенциально может привести к необходимости горизонтальной прокрутки для просмотра текста на всю его ширину

wbr - старело: использование этой функции больше не рекомендуется указывает браузеру, где можно сделать перенос строки в тексте, если этого требует ширина родительского элемента. Если какое-либо слово слишком длинное, или вы опасаетесь, что браузер будет делать переносы строк в ненадлежащем месте, то можно использовать тег wbr, чтобы жестко задать браузеру такое место переноса строки. Тег wbr был добавлен в HTML5. Устарело: использование этой функции больше не рекомендуется
